Notes

Each voter must provide a strict total order. Restrict the election to the Smith Set (in the sense of smith_set_rk), then run RuleIRV.

  • is_cm_():

    • cm_option = 'fast': Rely on RuleIRVâs fast algorithm. Polynomial heuristic. Can prove CM but unable to decide non-CM (except in rare obvious cases).

    • cm_option = 'slow': Rely on RuleExhaustiveBallotâs exact algorithm. Non-polynomial heuristic (\(2^{n_c}\)). Quite efficient to prove CM or non-CM.

    • cm_option = 'very_slow': Rely on RuleIRVâs exact algorithm. Non-polynomial heuristic (\(n_c!\)). Very efficient to prove CM or non-CM.

    • cm_option = 'exact': Non-polynomial algorithm from superclass Rule.

    Each algorithm above exploits the faster ones. For example, if cm_option = 'very_slow', SVVAMP tries the fast algorithm first, then the slow one, then the âvery slowâ one. As soon as it reaches a decision, computation stops.
